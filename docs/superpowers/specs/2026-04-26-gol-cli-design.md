# GOL CLI Tool — Design Spec

**Date:** 2026-04-26
**Status:** Approved
**Location:** `gol-tools/cli/`
**Language:** Go (Cobra framework)

## Overview

Unified command-line entry point for all GOL development operations. Replaces the scattered `shortcuts/` scripts, `.zshrc` aliases, and overlapping ai-debug features (boot/teardown/reimport) with a single cross-platform Go binary.

Core commands (run/stop/test/reimport) are native Go with no Node.js dependency. The `debug` command is a passthrough to the existing ai-debug Node.js tool.

## Command Tree

```
gol                                    # alias for `gol run game`
gol run game [--windowed] [--force]    # smart boot, headless by default
gol run editor                         # open Godot editor (detached)
gol stop [game|editor]                 # no args = stop all
gol test [unit|integration]            # 2-phase test runner, no args = both
gol reimport [--editor]                # headless reimport + UID ensure
gol debug <args...>                    # passthrough to ai-debug (requires Node.js)
gol install                            # build + symlink to PATH
gol help                               # auto-generated by Cobra
```

## Path Resolution

### Godot Binary

Resolution order:

1. `GODOT_PATH` env var
2. macOS: `/Applications/Godot.app/Contents/MacOS/Godot`
3. Windows: Steam paths → Scoop shim (`%USERPROFILE%\.local\bin\godot.bat`) → `C:\Program Files\Godot\` → `C:\Program Files (x86)\Godot\`
4. Fallback: `godot` on PATH

If not found: `"Error: Godot not found. Set GODOT_PATH or install Godot."`, exit 1.

### Project Path

Resolution order:

1. `--path` flag (available on all commands)
2. `GOL_PROJECT_PATH` env var
3. Auto-detect: walk up from CWD looking for `gol-project/` sibling or `project.godot` in current dir

## Command Details

### `gol run game [--windowed] [--force]`

Default = **headless**. Headless is faster, uses less resources, and supports `gol debug` commands (screenshot, console, eval) via the IPC bridge.

Smart boot sequence:

1. **Duplicate check** — find existing Godot game process for this project. If found and `--force` not set: print `"Game already running (PID <pid>). Use --force to start another or 'gol stop game' first."`, exit 1
2. **Ensure import cache** — if `.godot/` directory missing, run `godot --headless --import --path <project>`
3. **Launch** — `godot --headless --path <project>` (or without `--headless` if `--windowed`)
4. **Poll readiness** — watch for ai-debug signal directory (`~/Library/Application Support/Godot/app_userdata/God of Lego/ai_signals/`), poll every 100ms, timeout 30s
5. **Stream output** — pipe stdout/stderr to terminal AND timestamped log at `logs/game/game-YYYYMMDD-HHMMSS.log`

Exit code: game process exit code propagated.

Flags:
- `--windowed`: launch with GUI window instead of headless
- `--force`: skip duplicate-process check, allow multiple instances

### `gol run editor`

1. Launch `godot --editor --path <project>` as detached process
2. No multi-instance restriction
3. No headless option (editor requires a window)
4. Print PID, return immediately

### `gol stop [game|editor]`

| Invocation | Behavior |
|---|---|
| `gol stop game` | Kill only game processes (non-editor Godot with matching project path) |
| `gol stop editor` | Kill only editor processes (Godot launched with `--editor`) |
| `gol stop` | Kill all Godot processes for this project |

Process identification: match by process name + command-line args (`--editor` presence, `--path` value).

Kill sequence: SIGTERM → wait 5s → SIGKILL. Windows: `taskkill /PID` → `taskkill /F /PID`.

Output: `"Stopped Godot (PID <pid>)"` or `"No Godot process found"`.

### `gol test [unit|integration]`

No args = run both phases sequentially. Exit code: 0 if all pass, 1 if any fail.

**Phase 1 — gdUnit4 (unit):**

1. Clean `reports/` directory
2. Run: `godot --headless -s addons/gdUnit4/bin/GdUnitCmdTool.gd --add res://tests/unit/ --ignoreHeadlessMode --verbose`
3. Parse `reports/results.xml` (JUnit XML) using Go `encoding/xml`
4. Capture exit code

**Phase 2 — SceneConfig (integration):**

1. Discover: walk `tests/integration/`, grep for `extends SceneConfig` in each `.gd` file
2. Run each: `godot --headless --path . res://scenes/tests/test_main.tscn -- --config=res://tests/integration/<file>`
3. Capture exit code per file (0 = pass, non-zero = fail)

### `gol reimport [--editor]`

**Default (headless):**

1. Kill running Godot (same as `gol stop`)
2. Delete `.godot/imported/` directory
3. Run `godot --headless --import --path <project>`
4. Generate missing `.uid` files (walk project tree, create for `.gd`/`.tscn`/`.tres` without one)

**With `--editor`:**

1. Kill running Godot
2. Delete `.godot/imported/`
3. Launch `godot --editor --path <project>`
4. Wait for editor to finish scanning (poll `.godot/imported/` repopulation, timeout 120s)
5. Kill editor process

### `gol debug <args...>`

1. Check `node` on PATH — if not found: `"Error: Node.js is required for 'gol debug'. Install from https://nodejs.org"`, exit 1
2. Resolve ai-debug path: `<repo-root>/gol-tools/ai-debug/ai-debug.mjs` — if not found: `"Error: ai-debug not found at <path>. Is gol-tools submodule initialized?"`, exit 1
3. `exec node ai-debug.mjs <args...>` — replace process, full passthrough (stdin/stdout/stderr)

### `gol install`

1. `go build -o gol .` in `gol-tools/cli/`
2. Symlink to `/usr/local/bin/gol` (macOS/Linux) or copy to `%USERPROFILE%\.local\bin\gol.exe` (Windows)
3. Print success message

```
├── main.go                    # entry point, root command
├── go.mod                     # module: github.com/Dluck-Games/gol-cli
├── go.sum
├── cmd/
│   ├── root.go                # root command, global flags (--path, --verbose)
│   ├── run.go                 # `run` parent + `run game` / `run editor`
│   ├── stop.go                # `stop [game|editor]`
│   ├── test.go                # `test [unit|integration]`
│   ├── reimport.go            # `reimport [--editor]`
│   ├── debug.go               # `debug` passthrough
│   └── install.go             # `install`
├── internal/
│   ├── godot/
│   │   ├── binary.go          # Godot binary discovery
│   │   ├── process.go         # launch, kill, poll, duplicate detection
│   │   └── import.go          # headless import, cache clear, UID ensure
│   ├── project/
│   │   └── resolve.go         # project path resolution
│   ├── testrunner/
│   │   ├── runner.go          # orchestrator: discover → execute → report
│   │   ├── gdunit.go          # gdUnit4: invoke + parse JUnit XML
│   │   ├── sceneconfig.go     # SceneConfig: discover + invoke + parse
│   │   └── report.go          # ASCII report renderer
│   └── debug/
│       └── passthrough.go     # Node.js check + exec ai-debug
├── Makefile                   # build, install, clean
└── .goreleaser.yml            # cross-platform release config
```

### Dependency Boundaries

```
Core commands (run/stop/test/reimport)
  └── Godot binary only — no Node.js required

Debug command
  └── Node.js + gol-tools/ai-debug/ai-debug.mjs
  └── Graceful error if Node.js missing

godot-auto-uid.sh hook (post-migration)
  └── Calls `gol reimport` — no Node.js required
```
